Complete Reference

1

<select> Element

The <select> element creates a drop-down list (or scrollable list with multiple/size). It is a form-associated, labelable element and participates in constraint validation.

Content model In classic mode, permitted children are <option>, <optgroup>, and <hr>. With appearance: base-select opted in, the HTML parser is extended to also allow <button> (as the first child), <div>, <span>, <img>, <svg>, and most other elements inside <option> and as direct children.

<select> — Anatomy

<!-- Minimal -->
<select name="fruit" id="fruit">
  <option value="">Choose a fruit</option>
  <option value="apple">Apple</option>
  <option value="mango" selected>Mango</option>
</select>

<!-- Full attributes -->
<select
  id="my-select"
  name="my-select"
  form="my-form"
  multiple
  size="5"
  disabled
  required
  autofocus
  autocomplete="country-name"
  aria-label="Select your country"
  aria-required="true"
  tabindex="0"
>
  <option value="">-- Select --</option>
  <option value="in">India</option>
  <option value="us">United States</option>
</select>

<!-- With optgroup + hr divider -->
<select name="food">
  <option value="">Choose food</option>
  <hr>
  <optgroup label="Fruit">
    <option value="apple">Apple</option>
    <option value="mango">Mango</option>
  </optgroup>
  <hr>
  <optgroup label="Vegetables">
    <option value="carrot">Carrot</option>
  </optgroup>
</select>

<option> Element

Defines a single selectable item inside a <select>, <optgroup>, or <datalist>. In base-select mode the element can contain arbitrary HTML including images and SVG.

<!-- All option attributes -->
<option
  value="red"          <!-- value submitted with form -->
  label="Red Color"   <!-- visible label override -->
  selected            <!-- initially selected -->
  disabled            <!-- unselectable -->
>Red</option>

<!-- Rich content (base-select only) -->
<option value="html">
  <svg aria-hidden="true" width="16" height="16">…</svg>
  <span>HTML</span>
  <span class="desc">Markup language</span>
</option>

<!-- In datalist: value attribute is the suggestion -->
<option value="Firefox"></option>

<optgroup> Element

Groups related <option> elements inside a <select>. Cannot be nested. In base-select mode a <legend> element may be used as the group label instead of the label attribute, enabling full CSS styling of the heading.

<!-- Classic optgroup -->
<optgroup label="Swedish Cars">
  <option value="volvo">Volvo</option>
  <option value="saab">Saab</option>
</optgroup>

<!-- Disabled group -->
<optgroup label="Archived" disabled>
  <option value="old">Old Model</option>
</optgroup>

<!-- base-select: <legend> as styled label -->
<optgroup>
  <legend>🌿 Plant-Based</legend>
  <option value="tofu">Tofu</option>
  <option value="tempeh">Tempeh</option>
</optgroup>

<datalist> Element

<datalist> provides a list of predefined suggestions for <input> via the list attribute. It is not a replacement for <select> — the user can still type any value. Styling is browser-controlled and very limited.

<!-- Text autocomplete -->
<label for="browser">Browser:</label>
<input list="browsers" id="browser" name="browser">
<datalist id="browsers">
  <option value="Chrome"></option>
  <option value="Firefox"></option>
  <option value="Safari"></option>
  <option value="Edge"></option>
</datalist>

<!-- Color picker swatches -->
<input type="color" list="swatches">
<datalist id="swatches">
  <option value="#e84f1d"></option>
  <option value="#4a3aff"></option>
  <option value="#1d9e6f"></option>
</datalist>

<!-- Range tick marks -->
<input type="range" min="0" max="100" list="ticks">
<datalist id="ticks">
  <option value="0"></option>
  <option value="25"></option>
  <option value="50"></option>
  <option value="75"></option>
  <option value="100"></option>
</datalist>

<selectedcontent> New Chrome 135+

A new HTML element that, when placed inside the <button> child of a base-select <select>, automatically mirrors (clones) the inner HTML of the currently selected <option>. It enables you to show rich content (icons, descriptions) inside the closed select button — and to style it differently than the picker dropdown.

Progressive enhancement Non-supporting browsers completely ignore the <button><selectedcontent></selectedcontent></button> structure and fall back to the plain text of the selected option. No JS needed. 
<select name="tech" class="my-select">
  <!-- Custom button with cloned content -->
  <button>
    <selectedcontent></selectedcontent>
  </button>

  <option value="html">
    <span class="icon">🌐</span>
    <span class="name">HTML</span>
    <span class="desc">HyperText Markup Language</span>
  </option>
  <option value="css">
    <span class="icon">🎨</span>
    <span class="name">CSS</span>
    <span class="desc">Cascading Style Sheets</span>
  </option>
</select>

/* Hide the description inside the button but show it in the picker */
selectedcontent .desc { display: none; }
option .desc            { display: block; font-size: .75rem; color: #888; }

appearance: base-select New

The opt-in CSS value that unlocks the Customizable Select Element API. It must be set on both the <select> and its ::picker(select) pseudo-element. Setting it only on the <select> styles just the button; not setting it on the picker leaves the OS-rendered dropdown unchanged.

/* Minimum opt-in — apply to BOTH */
select,
::picker(select) {
  appearance: base-select;
}

/* Or with nesting */
select {
  appearance: base-select;
  &::picker(select) {
    appearance: base-select;
  }
}

/* Progressive enhancement check */
@supports (appearance: base-select) {
  select, ::picker(select) {
    appearance: base-select;
    /* custom styles here */
  }
}

What base-select unlocks:

::picker(select) ::picker-icon ::checkmark :open <button> first child <selectedcontent> Rich HTML in <option> Top-layer picker (popover) Anchor positioning @starting-style animation <legend> in <optgroup>
⚠ Limitations (as of early 2026) multiple and size attributes are not yet supported with base-select. Touch/mobile OS pickers are replaced by the web-rendered picker (no native mobile sheet). Only Chromium-based browsers (Chrome/Edge 135+) currently ship full support.

Picker Animations with @starting-style

Since ::picker(select) lives in the top layer with display:none when closed, normal CSS transitions don't work for the enter animation. The @starting-style at-rule (CSS Transitions Level 2) defines the initial style when the picker is first displayed — enabling enter transitions.

::picker(select) {
  appearance: base-select;

  /* Define target (resting) state */
  opacity: 1;
  transform: translateY(0);
  filter: blur(0);
  transition:
    opacity .2s ease,
    transform .2s cubic-bezier(.34, 1.56, .64, 1),
    filter .2s ease,
    overlay .2s ease allow-discrete,
    display .2s ease allow-discrete;

  /* Initial state when first shown */
  @starting-style {
    opacity: 0;
    transform: translateY(-10px) scale(.93);
    filter: blur(4px);
  }
}

/* Spring-bounce using cubic-bezier */
::picker(select) {
  transition: transform .35s cubic-bezier(.34, 1.56, .64, 1), opacity .2s;
  @starting-style { opacity: 0; transform: scale(.8) translateY(-12px); }
}

/* Slide from top */
::picker(select) {
  transition: transform .18s ease-out, opacity .15s;
  @starting-style { opacity: 0; transform: translateY(-16px); }
}

Accessibility Notes

✅ Always associate a <label> Use <label for="id"> or wrap the select inside the label. Never omit it — if you must, add aria-label or aria-labelledby.
📋 ARIA roles A native <select> maps to the combobox role. <option> maps to option. <optgroup> maps to group. These are correct by default — do not override unless you have a very good reason.
⚠ Rich content in base-select Adding non-text content (images, SVGs, spans) inside <option> that aren't hidden from the accessibility tree causes the picker's ARIA role to change from listbox to dialog. Always use aria-hidden="true" on decorative children, and ensure the option's accessible name is a clean text string. 
<!-- Correct: accessible rich option -->
<option value="js">
  <img src="js.svg" alt="" aria-hidden="true" width="16">
  <span>JavaScript</span>
  <span aria-hidden="true" class="desc">Scripting language</span>
</option>

<!-- aria-describedby hint text -->
<label for="lang">Language</label>
<select id="lang" aria-describedby="lang-hint" required>…</select>
<p id="lang-hint">Select the primary programming language for this project.</p>

📐 Progressive Enhancement Strategy Always write the base-select styles inside @supports (appearance: base-select) { … }. This ensures unsupported browsers continue to render the classic select while progressively gaining the rich custom UI in supporting browsers. The HTML markup is backwards-compatible: browsers that don't understand base-select strip unknown children from <option> and ignore the <button>/<selectedcontent> structure. 
/* Safe progressive enhancement pattern */
select {
  /* Classic styles for all browsers */
  appearance: none;
  border: 2px solid #ddd;
  border-radius: 6px;
  padding: .5rem 2.5rem .5rem 1rem;
  background: #fff url("arrow.svg") no-repeat right 1rem center;
}

@supports (appearance: base-select) {
  select, ::picker(select) {
    appearance: base-select;
  }

  select {
    /* Override classic styles */
    background: white;
    padding-inline-end: 1rem;
  }

  ::picker(select) {
    background: white;
    border: 1.5px solid #ddd;
    border-radius: 12px;
    box-shadow: var(--shadow-m);
    @starting-style { opacity: 0; transform: translateY(-6px); }

   ═══════════════════════════════════════════════════════ */

name           — form submission key
id             — for label association
disabled       — non-interactive
multiple       — allow multi-select (not with base-select)
required       — constraint validation
size           — visible rows (not with base-select)
autofocus      — focus on page load
form           — owner form id
autocomplete   — autofill hint token
tabindex       — focus order
aria-label     — accessible name

/* ═══════════════════════════════════════════════════════
   OPTION ATTRIBUTES
   ═══════════════════════════════════════════════════════ */

value          — submitted value (falls back to text content)
label          — display label override
selected       — pre-selected
disabled       — unselectable

/* ═══════════════════════════════════════════════════════
   OPTGROUP ATTRIBUTES
   ═══════════════════════════════════════════════════════ */

label          — required group heading
disabled       — disables all child options

/* ═══════════════════════════════════════════════════════
   CSS — PSEUDO-CLASSES
   ═══════════════════════════════════════════════════════ */

select:hover           select:focus            select:focus-visible
select:focus-within    select:active           select:disabled
select:enabled         select:required         select:optional
select:valid           select:invalid          select:user-valid
select:user-invalid    select:open             /* base-select only */
option:checked         option:disabled         option:hover
option:default         option:not(:checked)

/* ═══════════════════════════════════════════════════════
   CSS — ATTRIBUTE SELECTORS
   ═══════════════════════════════════════════════════════ */

select[multiple]       /* strip arrow, adjust padding for listbox */
select[size]           /* same — single-select listbox */
select[disabled]       /* same as :disabled */
select[required]       /* same as :required */
option[disabled]       /* unselectable option */
optgroup[disabled]     /* entire group disabled */
option[value=""]       /* placeholder / empty option */
option[selected]       /* initially selected option (HTML attribute) */

/* ═══════════════════════════════════════════════════════
   CSS — PSEUDO-ELEMENTS
   ═══════════════════════════════════════════════════════ */

::picker(select)     /* picker dropdown — base-select only */
::picker-icon        /* chevron arrow icon — base-select only */
option::checkmark    /* selected indicator — base-select only */
option::before       /* generated content — base-select only */
option::after        /* generated content — base-select only */

/* ═══════════════════════════════════════════════════════
   appearance PROPERTY VALUES
   ═══════════════════════════════════════════════════════ */

appearance: auto         /* OS native */
appearance: none         /* strip native, picker still OS */
appearance: base-select  /* full custom — Chrome/Edge 135+ */

/* ═══════════════════════════════════════════════════════
   ASSOCIATED ELEMENTS
   ═══════════════════════════════════════════════════════ */

<label>      — accessible name via for= or wrapping
<fieldset>   — group controls; disabled= disables all children
<legend>     — fieldset caption; base-select: optgroup label
<form>       — owner; select uses form= to associate off-page
<button>     — type=submit/reset/button; base-select: custom button child
<output>     — display computed / selected value (for=, name=)
<meter>      — scalar gauge (value, min, max, low, high, optimum)
<progress>   — task completion (value, max)
<kbd>        — document keyboard interactions in select UI guides

/* ═══════════════════════════════════════════════════════
   MINIMUM BASE-SELECT OPT-IN
   ═══════════════════════════════════════════════════════ */

select, ::picker(select) { appearance: base-select; }
