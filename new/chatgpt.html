web components + css states

Custom state pseudo-class: :state()

This is a very elegant bridge between custom elements and CSS: internal element states can become styleable without leaking everything into attributes or classes.

ElementInternals.states CustomStateSet :state(on)

Tap the custom element

off

When supported, the pill is styled by state-pill:state(on). Otherwise it falls back to a data attribute.

Why this is nice

  • You keep behavior inside the custom element.
  • You still get CSS-driven styling for state.
  • You avoid a lot of class toggling boilerplate.
class StatePill extends HTMLElement {
  #internals = this.attachInternals?.();
  toggle() {
    this.#internals?.states.add("on");
  }
}

state-pill:state(on) {
  /* style internal state */
}

spec parking lot

Interesting but still uneven: experimental selectors and newer modules worth watching

These are here because they matter conceptually, but they are not dependable enough yet for a broad live demo in every browser.

Column combinator ||

Useful for styling cells based on their table column relationship, but currently not a practical production selector.

/* Cells that belong to a selected column */
col.selected || td {
  background: color-mix(in oklab, var(--accent) 18%, canvas);
}

@position-try and overflow fallbacks

Part of the anchor-positioning family. The goal is to express fallback placements natively when tethered UI would overflow.

@position-try --flip-down {
  position-area: bottom span-inline-end;
}

.tooltip {
  position-try: --flip-down;
}

if() as a full value-level control structure

Already very interesting, but still a “use behind feature detection” tool for now.

.card {
  color: if(
    media(width > 900px): oklch(25% 0.04 280);
    else: currentColor;
  );
}

How to inspect this page

Open DevTools, watch the support matrix, resize the viewport, toggle the theme, open the popover, inspect the <select>, and shuffle the cards. Most sections are isolated so you can copy pieces into smaller experiments.

Built as a teaching lab, not a production starter kit. The whole point is to show what modern native platform primitives can do now — and which ones still need careful progressive enhancement.
