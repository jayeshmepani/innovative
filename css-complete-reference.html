02 — border-image

Border-image — the full 6-property family

border-image is its own 5-property system: border-image-source, border-image-slice, border-image-width, border-image-outset, border-image-repeat. The shorthand order is: source slice / width / outset repeat. Note: border-image overrides border-style visually but does not replace it in the cascade — you still need a border-width set.

box-shadow vs filter: drop-shadow() — the critical difference
box-shadow follows the box model — it always casts from the rectangular bounding box, even if the element uses border-radius, clip-path, or transparent PNG areas. filter: drop-shadow() follows the rendered alpha shape — it casts from the actual visible pixels. Prefer drop-shadow() for images, SVGs, and clipped shapes.
box-shadow on rectangle

follows box — rectangle

box-shadow: 10px 10px 20px rgba(0,0,0,.6);
filter: drop-shadow() on clip-path

follows alpha — polygon shape

filter: drop-shadow(6px 6px 12px rgba(251,146,60,.8));
/* drop-shadow() syntax (CSS Filters Level 1) */
filter: drop-shadow(offset-x offset-y blur color);
/* NO spread radius, NO inset keyword in drop-shadow() */

/* when to use each */
/* box-shadow:   UI cards, buttons, input fields, consistent box decorations */
/* drop-shadow: PNGs with transparency, SVGs, clip-path shapes, icons        */

07 — accessibility a11y

Focus rings — :focus-visible, forced-colors, keyboard patterns

WCAG 2.2 SC 2.4.11 requires focus indicators with at least 2px outline and sufficient contrast. Use :focus-visible (not :focus) so focus rings only appear for keyboard navigation, not mouse clicks.
three focus ring strategies — Tab or click each button
outline on :focus-visible (best practice)
outline: none;
:focus-visible {
  outline: 2px solid #38bdf8;
  outline-offset: 3px;
}

✓ WCAG-safe: visible for keyboard, hidden on mouse click

box-shadow ring on :focus-visible
:focus-visible {
  box-shadow:
    0 0 0 3px rgba(129,140,248,.4),
    0 0 0 6px rgba(129,140,248,.15);
}

⚠ box-shadow invisible in forced-colors / high-contrast mode

outline + forced-colors compatible

future properties — no browser support yet
border-shape Level 4
Allows arbitrary shapes for borders, separate from corner-shape. Enables non-rectangular border outlines.
border-limit
Limits how far a border extends around an element, enabling partial borders (e.g. only 50% of each side). Related to partial-border design patterns.
border-clip / border-clip-top/right/bottom/left
Clips portions of the border, enabling decorative gaps, half-borders, and interrupted border patterns without pseudo-elements.
box-shadow-color
Longhand for the color component of box-shadow. Part of Borders Level 4. Allows animating shadow color independently of other components.
box-shadow-offset
Longhand for offset-x and offset-y. Enables smooth individual-axis shadow animation.
box-shadow-blur
Longhand for the blur-radius component. Animate blur radius independently for subtle depth effects.
box-shadow-spread
Longhand for spread-radius. Part of the same Level 4 shorthand decomposition.
box-shadow-position
Controls inset vs outset independently from other components in the Level 4 grammar.
