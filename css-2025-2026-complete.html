::details-content — Animate details expand Chrome 131+ · FF 130+

What is CSS Cascade Layers?
@layer lets you define explicit cascade layers. Lower layers always lose to higher layers, regardless of specificity. Unlayered styles always win.
What is anchor positioning?
CSS anchor positioning lets you tether positioned elements to anchor elements via anchor-name and position-anchor. No JavaScript required for tooltips and dropdowns.
What is @scope useful for?
@scope lets you apply styles to a specific DOM subtree without leaking into siblings. Eliminates the need for BEM or CSS Modules for component isolation.
details::details-content {
  height: 0; overflow: hidden;
  transition: height 0.4s ease,
    content-visibility 0.4s allow-discrete;
}
details[open]::details-content { height: auto; }

override-colors — Font Palette Customization Baseline 2024

The override-colors descriptor allows surgical replacement of colors within a font's internal palette indices. Effectiveness depends on the font tech (COLR vs Raster) and matching the correct layer indices.

✅ Full Support
🎩 👔 👑
Indices: 0–4

Clothing/accessories use COLRv1 with low indices. These respond perfectly to brand-palette overrides.

⚠️ Partial Support
🧢 💼 👓
Indices: Varies

Some accessories work but may need different indices. Results vary by specific emoji design.

❌ Not Supported
🔥 🌸 🍎
CBDT/SBIX Bitmap

Natural objects use bitmap fallback or complex gradients. Colors are baked into pixels — cannot be overridden.

SVG
SVG glyphs in font
⚠️ Limited — opt-in
2. Why Hat (🎩) Works Specifically

The top hat emoji is rendered using COLRv1 with a simple 5-layer structure:

🎩 Top Hat Structure (COLRv1):
├── Layer 0: Hat crown (palette index 0)
├── Layer 1: Hat band (palette index 1)
├── Layer 2: Hat brim top (palette index 2)
├── Layer 3: Hat brim underside (palette index 3)
└── Layer 4: Background/accent (palette index 4)

Each layer references a color index in the CPAL (Color Palette) table, which CSS override-colors can modify.

3. Why Other Emojis Don't Work

Emojis like 🔥 (fire), 🌸 (cherry blossom), 🍎 (apple) fail because:

① CBDT Bitmap
Colors permanently baked into pixel data — no palette exists
② Different Indices
May use indices outside 0-4 range (15+, 25+, etc.)
③ Hardcoded Colors
Use 0xFFFF (foreground color) instead of palette indices
④ Gradient Paints
COLRv1 gradients don't respond to simple palette overrides
4. The Critical Limitation

From W3C CSS Fonts Module Level 4:
"If the font does not have a color at the specified palette index, that override is silently ignored."

CBDT Bitmap → No CPAL table → ❌ Ignored
Wrong Indices → Index mismatch → ❌ Ignored
Hardcoded → No palette ref → ❌ Ignored
